# 📚 books system (cyval)

The books system allows CyVal to search for books using the **open library api**, retrieve detailed book information, and store selected books in a **local database** so they can be reused later without repeatedly calling the external api.

This system acts as a bridge between the frontend and the open library api, converting external data into the application's internal `book` model.

# 🧠 How the books system works
```
frontend
│
▼
cyval backend (spring boot)
│
├── check local database
│
└── if book not stored
		        │
		        ▼
	    open library api
		        │
		        ▼
	    book data returned
		        │
		        ▼
	  optionally saved to database
```
# 🧠 Typical flow
  1. the frontend searches for a book
  2. the backend calls the open library api
  3. the backend converts results into `book` objects
  4. the user can choose to save a book
  5. the saved book is stored in the database
  6. future requests can retrieve it directly from the database
---

# 🗄️ Book database model

books are stored in the `books` table.

| field 	      | description 				                  |
|---------------|---------------------------------------|
| id 		        | primary key generated by the database |
| volumeid 	    | open library work or edition key 	    |
| title 	      | book title 				                    |
| authors 	    | comma-separated list of authors 	    |
| publisheddate | year the book was first published 	  |
| description 	| description of the book 		          |
| thumbnailurl 	| url of the book cover image 		      |
| isbn13 	      | isbn identifier if available 		      |

#### Example book object:

```json
{
  "id": 3,
  "volumeid": "/works/ol82563w",
  "title": "warriors: into the wild",
  "authors": "erin hunter",
  "publisheddate": "2003",
  "description": "first book in the warriors series...",
  "thumbnailurl": "https://covers.openlibrary.org/b/id/240726-m.jpg",
  "isbn13": "9780060525503"
}
```

# 🌐 API ENDPOINTS
Below, are all endpoints associated with books.

### 🔎 Search books (open library)

Searches open library for books matching a query.

### get /search-book
| parameter     |	description                           |
|---------------|---------------------------------------|
|q	            |               search query            |
|max	          | maximum results returned (default = 5)|

#### Example Request
```
http://localhost:8080/search-book?q=warriors%20into%20the%20wild&max=5
```
Returns a list of book objects retrieved from open library.

### 📖 Get book by volume id

Retrieves detailed information about a specific book.

#### get /book-volume
| parameter     |	description                           |
|---------------|---------------------------------------|
|volumeid	      |open library work or edition key       |

#### Example Request
```
http://localhost:8080/book-volume?volumeid=/works/ol82563w
```

### 💾 Save book to database

stores a book in the database using its open library key.

if the book already exists in the database, the existing record is returned instead of creating a duplicate.

#### post /books/savebyvolumeid
| parameter     |	description                           |
|---------------|---------------------------------------|
|volumeid	      |open library work or edition key       |

#### Example Request
```
http://localhost:8080/books/savebyvolumeid?volumeid=/works/ol82563w
```
Returns the saved book object from the database.

### 📚 Get all saved books
Retrieves every book stored in the database.

#### get /books
#### Example Request
```
http://localhost:8080/books
```
### 🔍 Find saved book by title

Searches the database for a saved book by its title.

#### get /book/name
| parameter     |	description                           |
|---------------|---------------------------------------|
|title	|exact book title|
#### Example Request
```
http://localhost:8080/book/name?title=warriors:%20into%20the%20wild
```
* Please note that underscores are able to work in place of "%20"

### 📚 Edit Books 
Editing book information manually.

#### PUT /books/{id}
#### Example Request
```
http://localhost:8080/books/3
```

#### Example Request Body
```
{
  "title": "warriors: into the wild (updated)",
  "authors": "erin hunter",
  "publishedDate": "2003",
  "description": "updated description of the first warriors book",
  "thumbnailUrl": "https://covers.openlibrary.org/b/id/240726-m.jpg",
  "isbn13": "9780060525503"
}
```

### 🗑️ Delete a saved book

Removes a book from the database.

#### delete /books/{id}

#### Example Request
```
http://localhost:8080/books/3
```
### 🔗 External api integration

CyVal uses the open library api to retrieve book metadata.

https://openlibrary.org/search.json

This api provides:

book titles

authors

cover images

isbn numbers

publishing dates

descriptions

The backend converts open library responses into book objects so they can be used consistently within the CyVal application.
