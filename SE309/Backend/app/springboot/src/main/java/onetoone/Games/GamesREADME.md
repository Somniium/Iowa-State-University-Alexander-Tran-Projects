# 🎮 games system (cyval)

The games system allows CyVal to search for video games using the **RAWG API**, retrieve detailed game information, and store selected games in a **local database** so they can be reused later without repeatedly calling the external API.

This system acts as a bridge between the frontend and the RAWG API, converting external data into the application's internal `Game` model.

---

# 🧠 How the games system works

```
frontend
│
▼
cyval backend (spring boot)
│
├── check local database
│
└── if game not stored
            │
            ▼
        RAWG API
            │
            ▼
        game data returned
            │
            ▼
      optionally saved to database
```

---

# 🧠 Typical flow

1. the frontend searches for a game
2. the backend calls the RAWG API
3. the backend converts results into `Game` objects
4. the user can choose to save a game
5. the saved game is stored in the database
6. future requests can retrieve it directly from the database

---

# 🗄️ Game database model

Games are stored in the `games` table.

| field | description |
|------|-------------|
| id | primary key generated by the database |
| rawgId | RAWG unique game identifier |
| name | game title |
| released | release date |
| rating | RAWG rating score |
| metacritic | metacritic score if available |
| backgroundImage | game cover / background image URL |
| website | official game website |
| genres | comma-separated list of genres |
| platforms | comma-separated list of platforms |
| description | detailed description of the game |

# 🌐 API ENDPOINTS

Below are all endpoints associated with games.

---

# 🔎 Search games (RAWG)

Searches the RAWG API for games matching a query.

### GET `/search-games`

| parameter | description |
|-----------|-------------|
| q | search query |
| max | maximum results returned (default = 10) |

### Example Request

```
http://localhost:8080/search-games?q=elden%20ring&max=5
```

Returns a list of lightweight `Game` objects retrieved from RAWG.

These are **search results only** and are **not saved to the database**.

---

# 🎮 Get game details from RAWG

Retrieves detailed information about a specific game.

### GET `/game-rawg`

| parameter | description |
|-----------|-------------|
| rawgId | RAWG game ID |

### Example Request

```
http://localhost:8080/game-rawg?rawgId=326243
```

Returns a detailed `Game` object including:

- description
- website
- metacritic score
- genres
- platforms

---

# 💾 Save game to database

Stores a game in the database using its RAWG ID.

If the game already exists in the database, the existing record is returned instead of creating a duplicate.

### POST `/games/saveByRawgId`

| parameter | description |
|-----------|-------------|
| rawgId | RAWG game ID |

### Example Request

```
http://localhost:8080/games/saveByRawgId?rawgId=326243
```

Returns the saved game object from the database.

---

# 📚 Get all saved games

Retrieves every game stored in the database.

### GET `/games`

### Example Request

```
http://localhost:8080/games
```

---

# 🔍 Find saved game by ID

Searches the database for a saved game using its database ID.

### GET `/games/{id}`

### Example Request

```
http://localhost:8080/games/3
```

---

# 🔍 Find saved game by name

Searches the database for a saved game by its title.

### GET `/games/name`

| parameter | description |
|-----------|-------------|
| name | exact game title |

### Example Request

```
http://localhost:8080/games/name?name=elden%20ring
```

---

# ✏️ Edit game information

Allows manual editing of game data.

### PUT `/games/{id}`

### Example Request

```
http://localhost:8080/games/3
```

### Example Request Body

```json

  "released": "2022-02-25",
  "rating": 4.5,
  "metacritic": 96,

```

---

# 🗑️ Delete a saved game

Removes a game from the database.

### DELETE `/games/{id}`

### Example Request

```
http://localhost:8080/games/3
```

# 🔗 External API Integration

CyVal uses the **RAWG API** to retrieve video game metadata.

```
https://api.rawg.io/api/games
```

This API provides:

- game titles
- cover images
- release dates
- ratings
- metacritic scores
- genres
- supported platforms
- official websites
- detailed descriptions

The backend converts RAWG responses into `Game` objects so they can be used consistently within the CyVal application.

---

# 🧩 Frontend Usage Flow

Typical frontend flow:

### 1️⃣ Search games

```
GET /search-games?q=elden ring
```

Display results as game cards.

### 2️⃣ User clicks a game

Use the `rawgId` from the search result.

### 3️⃣ Save the game

```
POST /games/saveByRawgId?rawgId=326243
```

The backend fetches the **full game details from RAWG** and saves them locally.

### 4️⃣ Load game page

Use the returned game object to render the detailed view.

---

# ⚠️ Notes

- Search results from RAWG are **lightweight** and may not include full metadata.
- Full game details are retrieved using the `/game-rawg` endpoint.
- The database prevents duplicate games by enforcing a unique `rawgId`.

# 📌 Summary

The games system allows CyVal to:

- search for games
- retrieve detailed game information
- store games locally
- support reviews and discussions for saved games

This enables CyVal to integrate video game discussion alongside other media types such as books, music, and movies.
